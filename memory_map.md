# 初版メモリマップ案

## 方針

ROM を最上位 2KB に配置し、その直下に I/O 領域を置く。こうすると ROM と I/O の衝突を避けつつ、上位アドレス側に周辺を集約できる。

## 推奨案

- RAM: `$0000-$1FFF` (8KB 最小構成)
- 予備 / 拡張 RAM: `$2000-$DFFF`
- I/O 領域: `$E000-$EFFF`
- ROM: `$F800-$FFFF` (2KB)

## MC6850 ACIA 配置案

- ACIA control/status: `$E000`
- ACIA data: `$E001`

この配置なら、今後ほかの周辺を追加する場合も `$E0xx` から順に割り当てしやすい。

## ROM を最上位に置く理由

- リセットベクタを含む最上位アドレス帯を素直に ROM へ割り当てられる
- 2KB ROM なら開始アドレスは `$F800` になり切りがよい
- 上位デコードが比較的単純になる

## ACIA を `$FFxx` に置かない理由

- ROM 領域と重なりやすい
- デコード回路が複雑化しやすい
- 将来の周辺追加でも管理しづらい

## 作業 RAM の想定

初版では、次のような RAM 使用を想定する。

- 入力行バッファ
- ロード用一時バッファ
- 16 進変換の作業領域
- 必要最小限の状態保持

合計で 1KB 程度を上限目安にする。

## ビルド時可変にしたい定義

- ROM base
- RAM start
- RAM end
- ACIA control/status address
- ACIA data address
- 行末判定種別

## 将来拡張

- ビデオ出力ビルドでは別の I/O マップ定義を使う
- 追加の周辺回路は `$E000-$EFFF` の範囲で管理する
- デバッグ支援回路を入れる場合も ROM と競合しないよう上位 I/O 領域に寄せる
