# 要件定義

## 目的

MC6800 向けに、少ない ROM / RAM で動作する ROM モニタを実装する。初版はシリアル端末経由の操作を前提とし、将来のビデオ出力対応やデバッグ機能拡張の土台にする。

## 対象環境

- CPU: MC6800
- 実機: 自作ボード
- シリアル I/O: MC6850 ACIA
- ACIA クロック: 153.6 kHz
- ボーレート: 9600 bps
- アセンブラ: Macro Assembler AS

## リソース制約

- ROM 目標サイズ: 2KB
- RAM 想定: 8KB から 64KB
- モニタ作業領域の目安: 1KB 程度

初版は 2KB ROM 制約を最優先し、後から追加できる機能は積極的に次フェーズへ回す。

## 機能要件

### 初版の必須機能

- 1 文字入力
- 1 文字出力
- 行入力
- バックスペース処理
- 16 進文字列入力
- エラー表示
- メモリダンプ
- メモリ書き換え
- 指定アドレスからの実行
- Motorola S-Record ロード
- Intel HEX ロード

### 初版で見送る機能

- S-Record 保存
- Intel HEX 保存
- レジスタ表示
- レジスタ変更
- ブレークポイント
- ステップ実行
- ビデオ出力

### 将来拡張

- ビデオ出力ビルド
- デバッグ支援機能
- ハード支援との連携
- フォーマット保存機能

## 互換性要件

- MIKBUG の全面互換は目指さない
- 電大版 BASIC が利用する文字入出力エントリーポイント互換を持たせる
- 少なくとも `INEEE` と `OUTEEE` 相当の呼び出し口を提供する
- `INEEE` は MIKBUG と同様に入力文字を `OUTEEE` へ渡してエコーバック可能な実装を優先する

## 入出力要件

- 文字コードは ASCII
- 入力行終端は既定で `CR`
- 出力行終端は `CRLF`
- 入力行終端条件はビルド時設定で変更可能にする

## 構成要件

- メモリマップは定義値で変更可能にする
- I/O アドレスは定義値で変更可能にする
- シリアル依存部とモニタ本体を分離する
- 将来のビデオ対応は同時対応ではなくビルド切り替えで対応する

## コマンド要件

コマンド体系は MIKBUG に寄せる必要はない。ユーザー向けコマンドは独自設計でよい。

初版候補:

- `D addr len`: メモリダンプ
- `M addr`: メモリ変更
- `G addr`: 指定アドレス実行
- `L SREC`: S-Record ロード
- `L IHEX`: Intel HEX ロード

## 設計指針

- 短いサブルーチンで構成し、再利用を優先する
- バッファは固定長で扱う
- 文字列処理は最小限に抑える
- フォーマットパーサは 16 進デコード処理を共有する
- エラーメッセージは短縮し ROM 消費を抑える

## 未確定事項

- 電大版 BASIC が期待する正確なエントリーポイント配置アドレス
- 実機の最終メモリマップ
- ACIA 以外の周辺回路追加方針
- デバッグ機能で使う割り込み経路
