# MC6800モニタに関する問答

* 想定ROM容量は何バイトですか。候補は 2KB / 4KB / 8KB あたりです。
→ 2k

* 想定RAM容量は何バイトですか。またモニタが占有してよい作業RAMはどの程度ですか。
→ 8kBから64kB(ROMで一部食われると想定) モニタバッファは1kB程度
→ 8kBはSBC6800ボードを想定している

* シリアルI/Oは何で実装しますか。MC6850 ACIA、ビットバンギング、別UART のどれを想定していますか。
→ MC6850 ACIA

* 文字端末の前提は ASCII のみでよいですか。制御コードは CR/LF と BS 程度で足りますか。
→ ASCIIのみ。入力はCRで行末判定、出力はCRLF。行末判定はビルド時に選択できるか設定できると良い

* S-Record と Intel HEX は、最初は ロードのみ でもよいですか。それとも最初から ロード/セーブ両方 必須ですか。
→ 最初はロードだけでOK。

* デバッグ機能は初期版から必要ですか。それともフェーズ分割して後回しでよいですか。
→ 次フェーズでOK。ハードも必要だと思う。

* ブレークポイントは ソフトウェアブレーク 前提でよいですか。MC6800の周辺回路側でNMI/IRQ支援がありますか。
→ ソフトブレークとハード支援両方を装備は盛りすぎ?? ハード支援を次フェーズでも良い

* 将来のビデオ対応では、シリアルとビデオの同時対応を目指しますか。それともビルド切替で十分ですか。
→ ビルド切り替え

* 使用するアセンブラは何を想定していますか。記法差があるので、対象を先に固定したいです。
→ http://john.ccac.rwth-aachen.de:8000/as/as_EN.html
→ Macro Assembler AS

* 実機ターゲットは自作ボードですか。既存評価ボードですか。メモリマップとI/Oアドレスが要件に直結します。
→ 自作ボード 。メモリマップとI/Oはデファインを調整してリビルドできる方法が良い

* 別件
  * MIKBUGの入出力エントリーポイント互換にできないだろうか。電大版BASICがそのまま動くのが理想。

---
1. MIKBUG互換は「完全互換」ではなく「BASICが使うI/O入口だけ互換」でよいですか。
→ OK

2. 電大版BASICが期待する具体的なエントリーポイント仕様は把握済みですか。資料があればそれに合わせて固定できます。
→ https://oando.web.fc2.com/old_mc6800/f100_501.htm#TAG042

3. ACIA の想定クロックとボーレートは何ですか。初期値が必要です。
→ 9600bps、ACIAへは153.6kHz

4. メモリマップ案はありますか。特に ROM配置アドレス、RAM先頭/末尾、ACIA制御/データレジスタアドレス は早めに決めたいです。
→ ROMは$FFFFから2kB、RAMは$0000から。ACIAはなるべく末尾に持っていきたい。ROMと競合すると駄目だよね、なんかいい案ある?

5. 初版コマンド体系は MIKBUG 風に寄せますか。それとも内部実装だけ互換で、ユーザー向けコマンドは独自でよいですか。
→ コマンドはMIKBUGに寄せる必要はない。文字入力、文字出力のエントリーポイントだけ互換にしたいだけ。
