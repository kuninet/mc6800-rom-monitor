# 実装計画

## 方針

初版は 2KB ROM 制約が厳しいため、ハード依存の最小 I/O 層から順に積み上げる。I/O、16 進変換、行入力などの共通部品を先に固め、後段のモニタ機能とローダで使い回す。

## 実装フェーズ

### フェーズ1: ACIA 最小層

目的:

- MC6850 ACIA で確実に 1 文字送受信できる土台を作る
- MIKBUG 互換 I/O 入口を先に固定する

成果物:

- ACIA 初期化
- 送信ルーチン
- 受信ルーチン
- ポーリング待ち
- `INEEE` / `OUTEEE` 相当の入口
- I/O アドレス定義

### フェーズ2: 対話基盤

目的:

- 端末からコマンドを受け取れる最小 CLI を作る

成果物:

- 1 行入力
- バックスペース処理
- 16 進文字デコード
- プロンプト表示
- 短いエラー表示

### フェーズ3: モニタ基本機能

目的:

- モニタとして最低限使える操作を揃える

成果物:

- メモリダンプ
- メモリ変更
- 指定アドレス実行
- コマンドディスパッチ

### フェーズ4: Hex ローダ

目的:

- 外部からプログラムを流し込めるようにする

成果物:

- S-Record ロード
- Intel HEX ロード
- チェックサム検証
- ロード結果表示

## Issue との対応方針

- 各フェーズは原則として別 Issue で管理する
- フェーズ内で実装量が大きい場合はサブ Issue に分割する
- 進捗の公式記録は Issue と PR に残す
- `docs/progress/` には日ごとの作業メモを残す
