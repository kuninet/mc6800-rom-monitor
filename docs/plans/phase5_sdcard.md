# 実装計画: フェーズ5 (SDカード SPI/FAT32 Read-Only 実装)

## 1. 目的と背景
SBC6800 に SBC-IO基板を接続し、MC6821 PIA のポートを用いたビットバンギングSPI通信により、SDカードからS-RecordやIntel HEXファイルを直接RAMへロードできる機能を実装する。
本計画は `docs/requirements/2026-04-25_sdcard_spi_fat_requirements.md` に基づき、ROM容量を節約しつつ、PC依存のシリアル転送から独立したローカルストレージからの起動手段（第一歩）を確立することを目的とする。

## 2. スコープ (対象範囲)
**初期実装 (本フェーズ) の範囲**
*   **ハードウェア/通信方式:** SBC-IO上のMC6821 PIA (Port B) によるSPI通信 (Mode 0)。
*   **対象メディア/ファイルシステム:** SDHCカード、FAT32、512バイトセクタ。
*   **ファイル/ディレクトリ制限:**
    *   Rootディレクトリの探索のみ (サブディレクトリ非対応)。
    *   8.3形式のファイル名のみ対応 (LFN非対応)。
    *   1ファイルの単発ロードのみ。
*   **操作(Read-Only):**
    *   `DIR` (または `F`) コマンド: Rootディレクトリのファイル一覧表示。
    *   `LF <filename>` コマンド: 指定したファイルの内容をStream入力とし、既存のローダ (S-Record/Intel HEX自動判別) に引き渡す。

**対象外 (別フェーズでの対応)**
*   SDカードへのSAVE（書き込み）機能。
*   SDSC、SDXC、exFAT、FAT16などの他フォーマット。
*   外付けマイコン (RP2040等) への処理の移譲（PoCとしての成立を優先するため、今回はPIA直結で行う）。
*   Boot Sector / Reserved Sector を利用した Bootstrap起動（今回はROM内に最小限のFAT処理を含める「ROM直FAT」を優先検証する）。

## 3. 実装ステップ

### Step 1: ハードウェア定義とエミュレータ拡張
1.  **`include/hardware.inc` の更新**
    *   PIAのベースアドレス（例: `$8020` 等、既存と衝突しないアドレス）と各レジスタ（CRA, DDRA, PRA, CRB, DDRB, PRB）の定義を追加。
    *   SPI信号用ビットマップ定義（MOSI, MISO, SCLK, CSなど）を追加。
2.  **`emu/sbc6800_emu.py` の機能拡張**
    *   仮想的なPIAクラスを実装し、指定アドレスへのアクセスをハンドリングする。
    *   疑似的なSDカードイメージ（ディスクイメージファイル）を読み込み、SPIコマンド（`CMD0`, `CMD8`, `ACMD41`, `CMD58`, `CMD17`）に応答する状態遷移マシンを実装する。

### Step 2: SPIドライバ・SD初期化ルーチンの実装 (アセンブリ)
1.  **PIA初期化 (`pia_init`)**: Port Bの入出力方向設定。
2.  **1バイト送受信 (`spi_xfer`)**: SCLKを操作しながら、MOSIへの1ビット出力とMISOからの1ビット入力を8回ループする最小単位。
3.  **SDカード初期化 (`sd_init`)**: ダミークロック送出 → `CMD0` → `CMD8` → `ACMD41` のシーケンス実装。
4.  **1セクタ読み出し (`sd_read_sector`)**: `CMD17` を発行し、512バイトのデータを指定バッファ (RAM) へ読み込む。

### Step 3: FAT32 (Read-Only) ルーチンの実装
1.  **BPB（BIOS Parameter Block）の読み取り**: セクタ0 (またはパーティション先頭) からBPBを読み取り、RootディレクトリエリアやFATの開始セクタを特定する。
2.  **Rootエントリ走査 (`sd_dir`)**: `DIR` コマンド向け。Rootディレクトリエリアの各32バイトエントリを読み取り、削除済みやLFNを除外して8.3ファイル名を表示する。
3.  **ファイルオープン・Stream読み出し (`sd_load_file`)**: `LF` コマンド向け。指定ファイル名から開始クラスタを特定し、FATチェインをたどりながらセクタ単位で読み出し、1バイトずつ既存のローダ（`READ_RECORD`相当）へ文字を供給する。

### Step 4: モニタコマンドの統合とテスト
1.  `src/main.asm` のコマンドディスパッチャに `DIR` と `LF` を追加。
2.  エミュレータ上で FAT32 形式のダミーイメージを使用し、`DIR` と `LF TEST.S` が正常に機能することを確認（スモークテストの追加）。

## 4. RAM / ROM マップの検討（調整）
本フェーズでは PIA 直結の検証を行うため、以下の構成をベースとする。
*   **ROM ($E000-$FFFF)**: SD/FAT の基本ルーチンをこの2KB(または拡張した4~8KB)に収める。入り切らない場合は、SBC-IOのRAM拡張(`$C000-$DFFF`)を前提とし、一部をロードするBootstrap方式への切り替えを検討する。
*   **セクタバッファ**: 512バイトのバッファを、SBC-IOの拡張RAM（例:`$C000`〜）または既存のモニタ用作業領域に配置する。
