# 進捗ログ 2026-04-08

## 実施内容

- 管理者 PowerShell で TL866II Plus を `usbipd bind --busid 2-3` した
- 通常 PowerShell から WSL2 への `attach` を試し、Windows 側占有で失敗するケースを確認した
- WSL 側で `Ubuntu 22.04` の `apt` に `minipro` パッケージが無いことを確認した
- `minipro` をソースからビルドして導入した
- `udev/60-minipro.rules` を `/etc/udev/rules.d/` へ配置し、`plugdev` グループ設定を反映した
- WSL 側で `lsusb` に TL866II Plus が見えることを確認した
- WSL 側で `minipro --version` が `Found TL866II+` を返すことを確認した
- `make program-w27c512` を実行し、`W27C512@DIP28` への消去、書き込み、検証成功を確認した

## 判明したこと

- この環境では `usbipd-win + WSL2 + minipro` の経路で TL866II Plus を実用できる
- `Ubuntu 22.04` では `minipro` を APT から直接入れられず、ソースビルドが必要だった
- `attach` 失敗時の `Device busy (exported)` は Windows 側の占有が原因で起きる
- `W27C512` では書き込み時間が約 `29.87` 秒、読み出し検証が約 `1.11` 秒だった

- `verify` と `readback` ターゲットも WSL 側で確認する
- `27C64` と `27C256` 系の書き込み可否も順に確認する
